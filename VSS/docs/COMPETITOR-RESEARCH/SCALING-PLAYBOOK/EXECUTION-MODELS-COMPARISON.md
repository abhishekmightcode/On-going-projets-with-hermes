# Solar Execution Models — How Companies Deliver Installations
**Comparison of execution approaches among Bangalore and national competitors**

---

## What is an "Execution Model"?

An execution model describes how a solar company goes from a signed deal to a commissioned, generating solar rooftop — covering:
- Site assessment
- System design
- Material procurement
- Installation labor
- Quality inspection
- Net metering / grid connection
- Handover to customer

---

## Model A: Traditional Sales → Sub-Contractor Execution

**Used by:** Smaller local solar companies, dealers

**Flow:**
```
Lead → Sales visit → Quote → Deal → Company orders material
→ Subcontracted installer dispatched → Customer site
→ Company inspects → Hands over
```

**Who does what:**
- Company: Sales, design, material procurement, quality
- Subcontractor: Installation labor, mounting
- Customer: Pays, waits

**Characteristics:**
- Low overhead
- Quality depends on subcontractor skill
- Communication gaps between sales promise and execution reality
- Slower — depends on sub's schedule
- Margin: 25–35%

---

## Model B: In-House Team + Project Lead (VSS Model)

**Used by:** VSS, EcoSoch, established local EPCs

**Flow:**
```
Lead → Site visit by Project Lead → Design + Quote
→ Deal signed → Material dispatched to site
→ Project Lead oversees installation
→ In-house technician team executes
→ Project Lead does QC → Commissioning → Handover
```

**Who does what:**
- Company: Everything in-house (sales support, design, procurement, execution)
- Project Lead: Single point of ownership for project delivery
- In-house technicians: Installation, electrical work
- Customer: Pays, gets one point of contact

**Characteristics:**
- Higher overhead (salaries, tools, vehicles)
- Better quality control
- Faster communication
- Single person accountability (Project Lead)
- Margin: 30–45%

---

## Model C: Centralized Dispatch + City Hub (ZunSolar Model)

**Used by:** ZunSolar, large DTC brands

**Flow:**
```
Lead → App/website inquiry → Central design team (remote)
→ City Hub receives order → Local partner assigned
→ Partner sends own team → Installation
→ Central QC (remote check via IoT) → Commissioning
```

**Who does what:**
- Company (HQ): Lead capture, design, tech (IoT), quality standards
- City Hub: Partner coordination, material staging
- Local Partner: Installation labor (often uses own technicians)
- IoT: Remote monitoring from day 1

**Characteristics:**
- Technology replaces some in-house capacity
- Partners are semi-independent (franchise-style)
- Speed is competitive advantage
- Quality varies by partner city
- Margin: 20–30% (shared with partner)

---

## Model D: Government / B2G Project Model

**Used by:** Tata Power Solar, Adani Solar, Vikram Solar

**Flow:**
```
RFP / Tender → Technical bid → Financial bid
→ Project awarded → Design + BoQ preparation
→ EPC contract → Third-party construction management
→ Progress milestones → Government inspection
→ Commissioning → Defect liability period (12–24 months)
```

**Who does what:**
- Company: Tender writing, design, procurement, project management
- Third-party contractor: Physical installation
- Government: Inspections, payments (delayed)

**Characteristics:**
- Large project sizes (₹5–100 crore per project)
- Long payment cycles (12–36 months)
- Requires working capital / bank guarantee
- Margin: 12–18%
- Not suitable for residential / SMB

---

## Model E: Society / RWA Group Model

**Used by:** Companies targeting apartment complexes

**Flow:**
```
RWA approached → Society presentation → Member voting
→ Group deal negotiated (discount for volume)
→ Individual home owner agreements signed
→ Bulk material order → Sequential installation per flat/house
→ Individual net meters → AMC offered
```

**Who does what:**
- Company: RWA pitch, group negotiation, individual agreements, logistics
- RWA: Internal communication, collection, decision facilitation
- Individual owners: Sign, pay, manage own meter

**Characteristics:**
- Higher volume from single sale effort
- Long sales cycle (3–9 months for RWA decision)
- Complex logistics (individual owner coordination)
- AMC revenue stream per unit
- Margin: 25–40% (slightly lower per unit but volume)

---

## Model F: Hybrid — Solar + Storage / EV

**Used by:** Forward-thinking residential/commercial installers

**Flow:**
```
Lead → Site assessment (roof + load profile)
→ Solar + Battery sizing (hybrid system)
→ Deal signed → Battery + panel procurement
→ Solar + inverter + battery installation
→ Commissioning → Customer educated on system
```

**Who does what:**
- Company: Solar + battery system design, battery procurement, integration
- Customer: Pays premium for energy independence

**Characteristics:**
- Higher ticket size (battery adds ₹1–5 lakh)
- Differentiated offering
- Customer is more sophisticated (understands value)
- Fewer competitors in this space
- Margin: 35–50% (battery has higher margin)

## Key Insight: Execution is the Moat

For a company like VSS, the **execution model is the competitive moat**. The differences that matter:

1. **Who owns the project end-to-end?** (Project Lead = VSS's approach)
2. **How fast can you commission?** (VSS should target 30 days max)
3. **How does the customer feel during installation?** (Clean site, communicated schedule, professional team)
4. **What happens after?** (AMC, monitoring, relationship)

VSS's Executionflow tool is already ahead of most Bangalore competitors — it tracks:
- GPS location
- Filter stages
- n8n webhook → Zoho CRM

**VSS should document and market its execution discipline as a differentiator.**

## VSS Execution Best Practices (from research)

### What top performers do:
1. **Pre-site assessment** before quoting (shadow profiling, roof measurement, load analysis)
2. **Fixed timeline commitment** in the agreement (with milestone penalties)
3. **Photo documentation** at every stage (foundation → mounting → panel → wiring → meter)
4. **Single Project Lead** as customer WhatsApp contact
5. **Daily update WhatsApp broadcast** to customer during installation
6. **Clean site protocol** — tools and materials organized, site cleaned daily
7. **Formal handover** — walk-through with customer, meter readings, document packet
